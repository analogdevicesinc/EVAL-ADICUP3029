Detailed Description

Non-RTOS Framework Functions.

The set of functions in this module provide an application with a a simple interface for waiting for events from the radio with a timeout. This module is for applications in a non-RTOS operating environment. This layer uses a General Purpose Timer to implement timeouts, the user can configure the timeout in the adi_ble_config.h configuration header file.

ADI_BLER_RESULT adi_ble_DispatchEvents ( const uint32_t  nTimeoutMilliSeconds)

Event Dispatcher.

Dispatch any events received for a fixed amount of time to the application callback. This function will block the caller for the specified amount of time, and pass events to the callback.

Parameters
[in]nTimeoutMilliSeconds: The time to wait (ms) if no event occurs before erroring out.
Returns
ADI_BLER_RESULT
  • ADI_BLER_SUCCESS is returned upon success
  • ADI_BLER_FAILURE is returned upon failure

const uint32_t  nTimeoutMilliSeconds 
)

Wait for Event with Timeout.

This function will block until eDesiredEvent occurs. If there is a duration of nTimeoutMilliSeconds between calling this function and the desired event, the function will timeout and return failure. If a different event is received (not eDesiredEvent) it is passed to the application supplied callback.

Parameters
[in]eDesiredEvent: The event we are expecting to receive.
[in]nTimeoutMilliSeconds: The time to wait (ms) if no event occurs before erroring out.
Returns
ADI_BLER_RESULT
  • ADI_BLER_SUCCESS is returned if eDesiredEvent is received prior to timeout.
  • ADI_BLER_FAILURE is returned if eDesireEvent is not recevied prior to timeout, or a timer error occurs.
